  <p>The Foundation’s staff support MariaDB Server by:</p>
  <ul>
  	<li> Encouraging adoption, contribution and distribution of MariaDB Server and related open source software.</li>
  	<li> Helping new contributors understand the source code and lowering the barrier for new participants.</li>
  	<li> Participating in the design of new features, ensuring the code quality and maintainability of the codebase while the number of contributors is growing.</li>
  	<li> Making sure documentation keeps up with the development of new features.</li>
  	<li> Porting and packaging MariaDB to different platforms and operating systems to keep it as widely available as possible.</li>
  	<li> Speaking at events and driving adoption.</li>

  <h2>MariaDB Foundation in 2022</h2>
  <p>MariaDB Server 10.6 was released as GA in July 2021.</p>
  <p>Highlights include:<p>
  <ul>
    <li> Improved readability for DDL (Data Definition Language) operations to make most of them atomic, and the rest crash-safe, even if the server crashes in the middle of an operation.</li>
    <li> Adding SELECT ... SKIP LOCKED syntax for InnoDB only that causes those rows that couldn't be locked to be excluded from the result set.</li>
    <li> Ignored indexes which are indexes that are visible and maintained, but which are not used by the optimizer.</li>
    <li> Implement SQL-standard Offset and Fetch clauses for the SELECT expression.</li>
    <li> Optimizations that speed up inserts into an empty table</li>
    <li> Increase the maximum length of the MASTER_HOST string from 60 characters in MariaDB 10.5, to 255 characters in MariaDB 10.6.</li>
    <li> Allow transition from unencrypted to TLS Galera cluster communication without cluster downtime</li>
  </ul>
  <p>You can read more about MariaDB Server 10.6 <a href="https://logos-download.com/wp-content/uploads/2016/02/Intel_Logo_2020.png">here</a>.</p>
  <p>
    In 2021 MariaDB Server 10.7 and MariaDB Server 10.8 launched having a new release model. The new release model aims to allow users to preview the main features of a release before it is incorporated in the stable (GA) release. In the meantime users can test the features, send bug reports and give feedback to MariaDB. Some notable featured for the two releases are:
  </p>
  <ul>
    <li>MariaDB Server 10.7
      <ul>
        <li>UUID Data Type</li>
        <li>Natural Sort</li>
        <li>Compression Provider Plugins</li>
        <li>JSON Histograms</li>
        <li>Python-like String Formatting</li>
        <li>CONVERT PARTITION</li>
        <li>Password Reuse Check Plugin</li>
        <li>InnoDB Bulk Insert</li>
      </ul>
    </li>
    <li> MariaDB Server 10.8
      <ul>
        <li>IN, OUT, and INOUT/IN OUT parameters in CREATE FUNCTION</li>
        <li>InnoDB redo log improvements</li>
        <li>Descending indexes</li>
        <li>Auto create partition</li>
        <li>Windows – Improved i18n support</li>
        <li>Lag free ALTER TABLE in replication</li>
      </ul>
    </li>
  </ul>
  <p>Read more about all the features included in <a href="https://mariadb.com/kb/en/changes-improvements-in-mariadb-107/">MariaDB Server 10.7</a> and <a href="https://mariadb.com/kb/en/changes-improvements-in-mariadb-108/">MariaDB Server 10.8</a>.</p>
